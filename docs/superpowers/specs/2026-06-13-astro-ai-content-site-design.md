# Astro AI Content Site Design

## Purpose

Build `story.dbb.tw` as a separate reading site for completed AI-generated
articles. The home page must clearly disclose that the site's articles are
generated by AI and direct visitors who want human-written content to
`https://dbb.tw`.

The site should feel distinct from the Hugo PaperMod site at `dbb.tw`. Astro
Theme Pure provides the visual design and initial components. The project will
preserve the theme's reading-focused style and customize only the features
required by this site's publishing workflow.

## Scope

The first version includes:

- A home page that discloses the site's AI-generated content, links to
  `https://dbb.tw`, and shows a configurable number of recent posts.
- A complete Blog index.
- Individual Blog post pages.
- A Tags index and tag archive pages.
- Static output deployable to Netlify or Vercel.
- A README that defines how another LLM must deliver posts.

The first version does not include:

- Categories or category archive pages.
- A separate About page.
- Draft or review states. Every delivered post is ready to publish.
- Author archive pages.
- Translated copies of the same post.
- Tag translation mappings.
- An enabled comment system.

## Theme

Use Astro Theme Pure as the project foundation.

The theme should retain its reading and publication-oriented visual identity so
that `story.dbb.tw` does not resemble the PaperMod-based personal site at
`dbb.tw`.

Only required changes should be made:

- Adapt the Blog collection to the folder-per-post delivery format.
- Make a folder's `index.md` route to `/blog/<folder-name>` without a trailing
  `/index`.
- Require a local cover image for every post.
- Preserve the Tags index and tag archive pages.
- Make the number of recent home-page posts configurable.
- Replace the template home-page introduction with the AI-content disclosure
  and link to `https://dbb.tw`.
- Remove or disable features that conflict with the agreed publishing flow.

## Pages And Navigation

Primary navigation contains:

- Home
- Blog
- Tags

The home page contains:

- A clear statement that articles on `story.dbb.tw` are generated by AI.
- A prominent link telling visitors to visit `https://dbb.tw` for
  human-written content.
- The newest Blog posts, limited by a central site setting that defaults to
  five.

The Blog index lists every published post from newest to oldest. The Tags index
lists all tags, and each tag archive lists posts containing that tag.

## Content Architecture

Each post is a self-contained directory under `src/content/blog/`.

```text
src/content/blog/
└── tai-bei-lu-xing/
    ├── index.md
    ├── cover.jpg
    └── street.jpg
```

The directory name:

- Must use a stable URL-safe English or pinyin name.
- Becomes the post URL: `/blog/tai-bei-lu-xing`.
- Must not change after publication unless a URL migration is intentional.

The `index.md` filename is an implementation detail and must never appear in the
public URL.

Markdown references post images with relative paths:

```md
![台北街景](./street.jpg)
```

Astro Content Collections validate every post during the build. A post missing
required metadata or its cover image must fail the build.

## Post Metadata

Every post requires:

```yaml
---
title: 台北旅行
description: 文章摘要
publishDate: 2026-06-13
language: zh-TW
tags:
  - 台灣
  - 台北
heroImage:
  src: ./cover.jpg
  alt: 台北街景
---
```

Optional metadata:

```yaml
updatedDate: 2026-06-15
```

Rules:

- `publishDate` controls sorting from newest to oldest.
- `updatedDate` is displayed on the post but does not affect list ordering.
- `tags` contains one or more tags.
- Tag values may be Chinese or English.
- Tags are normalized according to the site's content schema and used as their
  URL segments.
- `language` identifies the post language; Traditional Chinese is the default,
  but posts in other languages are accepted.
- `heroImage.src` must reference a local image in the post directory.
- `heroImage.alt` is mandatory.
- Category metadata is not used.
- Draft and comment metadata are not part of the external content delivery
  format.

## Content Delivery Contract

Another project or LLM may create and update content only inside
`src/content/blog/`.

The content delivery README must tell an LLM to:

1. Create one complete directory per post.
2. Use a stable English or pinyin directory name.
3. Put `index.md`, the required cover, and all article images together.
4. Use the exact required frontmatter fields.
5. Add one or more useful tags.
6. Reference images using relative paths.
7. Deliver only completed, publishable content.
8. Run the documented validation command before finishing.

The README must prohibit an external content-producing LLM from modifying:

- Site source code and components.
- Site configuration.
- Theme files.
- Existing posts unrelated to its task.

## Site Configuration

A central configuration module stores at least:

```text
siteTitle
siteDescription
siteUrl
defaultLanguage
homePostLimit
```

The initial `homePostLimit` is `5`.

## Comments

Astro Theme Pure's Waline integration remains disabled in the first version.
The site must work without a Waline server URL.

Waline may be enabled later and shared with the Hugo site at `dbb.tw`. If both
sites share one Waline deployment, comment paths must include a site namespace
to prevent collisions:

```text
/dbb/...
/story/...
```

## Error Handling And Validation

Astro's content schema is the primary validation boundary.

The build must fail when:

- A required frontmatter field is missing or invalid.
- A post has no local cover image or cover alt text.
- A referenced local image cannot be resolved.
- A post directory creates a duplicate route.
- A generated Blog route contains an unwanted `/index` segment.

Deployment must proceed only after a successful production build.

## Testing

Initial verification includes:

- A production build.
- Content schema validation using valid and invalid fixture posts.
- Route generation checks for Blog posts and tags, including Chinese tag
  values.
- A route check proving that `src/content/blog/example/index.md` generates
  `/blog/example` and not `/blog/example/index`.
- Manual browser checks of the home page, Blog index, Tags pages, and post
  pages.
- A check that the home page clearly discloses AI-generated content and links
  to `https://dbb.tw`.
- A check that the site builds successfully while Waline is disabled.

## Deployment

The site uses Astro static output and remains compatible with both Netlify and
Vercel. No platform-specific adapter is required for the first version.
